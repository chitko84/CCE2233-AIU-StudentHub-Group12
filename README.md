## 3. Project Overview

The AIU Student Hub is a centralized academic and administrative platform used by students, lecturers, and management/staff at Albukhary International University. The system supports activities such as attendance, course registration, academic records, examination results, announcements, SRC elections, and communication between students and university departments.

During the requirement elicitation process, the group identified several challenges with the current Student Hub, including fragmented services, repeated login requirements, outdated user experience, limited notification support, manual report generation, scattered student information, and security/privacy concerns.

This project applies Requirements Engineering practices to collect, analyze, document, prioritize, validate, and manage requirements for an improved AIU Student Hub system.

---

## 4. Project Objectives

The main objectives of this project are:

1. To gather requirements for the AIU Student Hub using suitable requirement elicitation techniques such as surveys, interviews, observation, and brainstorming.
2. To document functional and non-functional requirements clearly and systematically.
3. To prioritize requirements using the MoSCoW prioritization technique.
4. To maintain requirement traceability using a Requirements Traceability Matrix (RTM).
5. To record and manage changes to requirements through version control and change management.
6. To apply ethical practices during stakeholder engagement, including informed consent, privacy protection, confidentiality, and fair handling of stakeholder input.

---

## 5. Stakeholder Groups

The project focused on three major stakeholder groups:

### 5.1 Students

Students are the primary users of the AIU Student Hub. They use the system to access academic and student-related services such as attendance records, examination results, course registration, timetables, announcements, SRC elections, and academic progress information.

Main concerns identified from students include:

* Repeated login requirements
* Outdated user interface and user experience
* Difficulty accessing attendance records
* Lack of timetable and class cancellation notifications
* SRC election security concerns
* Need for centralized access to academic information
* Privacy and security concerns

### 5.2 Lecturers

Lecturers use the Student Hub and related university systems to support teaching, attendance tracking, academic advising, reporting, and student progress monitoring.

Main concerns identified from lecturers include:

* Multiple disconnected systems
* Lack of unified student profiles
* Difficulty accessing advising records
* Manual academic report generation
* Timetable conflict issues
* Limited visibility into student academic risks
* Need for secure and role-based access to student information

### 5.3 AIU Management and Staff

Management and staff use the Student Hub for monitoring student records, academic performance, attendance, reports, internship tracking, and administrative decision-making.

Main concerns identified from management/staff include:

* Dashboard and reporting needs
* Role-based access control
* Audit logs
* Internship management
* QR attendance tracking
* Student ID data linking
* Data integrity
* Privacy and security
* System integration

## 6. Requirement Elicitation Methods

The group used multiple requirement elicitation methods to improve the reliability of the collected data.

| Requirement Category | Elicitation Method                               | Purpose                                                     |
| -------------------- | ------------------------------------------------ | ----------------------------------------------------------- |
| Satisfier            | Student survey and stakeholder interviews        | To identify expected and useful system features             |
| Dissatisfier         | Observation of the current AIU Student Hub       | To identify problems and limitations in the existing system |
| Delighter            | Brainstorming and future improvement discussions | To identify potential enhancements and future features      |

The use of multiple methods helped the group compare findings from different sources and reduce the risk of relying on only one type of stakeholder input.

## 13. Ethical Considerations

Ethical practices were important throughout this project because the group collected data from real stakeholders. The following ethical measures were applied:

* Interview participants were informed about the purpose of the project.
* Participants were asked to provide consent before interviews were recorded.
* Participation was voluntary.
* Participants could refuse to answer questions or withdraw.
* Personal information was handled carefully.
* Survey responses were collected without unnecessary personal identifiers.
* Sensitive evidence such as recordings, transcripts, and consent forms was stored in restricted folders.
* Stakeholder responses were used only for academic purposes.
* Suspicious AI-generated survey responses were excluded from requirement analysis to protect data validity.
* The group considered fairness by comparing student, lecturer, and management perspectives before prioritizing requirements.

## 16. Prioritization Method

The project used the **MoSCoW prioritization method**.

| Priority       | Meaning                                                                       |
| -------------- | ----------------------------------------------------------------------------- |
| Must Have      | Critical requirement needed for the system to function properly               |
| Should Have    | Important requirement that improves the system but is not absolutely critical |
| Could Have     | Useful enhancement that can be added if time and resources allow              |
| Won’t Have Now | Requirement excluded from the current version                                 |
